# Atlassian Skill

Jira + Bitbucket operations via `atlassian-cli`. No other tools.

## Guardrails

1. **`atlassian-cli` only** — never `gh`, `glab`, or Atlassian MCP tools for any Jira or Bitbucket operation.
2. **Format split** — Jira descriptions require ADF JSON. Bitbucket PR descriptions use plain Markdown. Never swap them.
3. **Transitions** — never hardcode a status name. Always fetch available transitions first, then apply by exact name.

## Bootstrap: Resolve Project Coordinates

Before any `atlassian-cli` command, resolve coordinates via this priority chain:

```
.atlassianrc (cached) → project CLAUDE.md vars → auto-detect → prompt user → write .atlassianrc
```

### Step 1 — Check for `.atlassianrc`

If `.atlassianrc` exists in the repo root, read it and use those values. Skip to commands.

### Step 2 — Auto-detect

```sh
# Workspace + repo slug from git remote
git remote get-url origin
# Expected shape: https://bitbucket.org/<workspace>/<repo>.git
# or:             git@bitbucket.org:<workspace>/<repo>.git

# Jira project key — scan branch names for issue key pattern
git branch -a | grep -oE '[A-Z]+-[0-9]+' | head -20
# Extract the prefix (e.g. "LRIS" from "LRIS-188")
```

### Step 3 — Confirm with user

Always confirm the detected Jira project key before first use:

```
Detected Jira project key: LRIS (from branch "LRIS-188-feature")
Confirm? [LRIS] or enter a different key:
```

Accept enter = use default. Any other input = use that value.

### Step 4 — Write `.atlassianrc`

```yaml
# Auto-generated by atlassian skill. Add to .gitignore.
workspace: <bitbucket-workspace>
repo: <repo-slug>
jira_project: <PROJECT-KEY>
jira_base_url: https://<workspace>.atlassian.net
detected_from: git-remote
last_updated: <YYYY-MM-DD>
```

Add `.atlassianrc` to `.gitignore` if not already present.
